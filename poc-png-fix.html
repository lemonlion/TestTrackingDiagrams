POC: PNG Export (No Transparency) Fix

This tests the "Copy/Save as PNG (no transparency)" feature for PlantUML diagrams rendered client-side via plantuml-js. Each test case renders a diagram, then exports it to PNG and checks the corner pixels for transparency.

Fix Approach Comparison

For each diagram, we compare four approaches:

  1. Current (broken): Uses getBackgroundColor(svg) which falls through to first visible rect fill — picks up diagram element colours (grey participant boxes)
  2. Fix A: SVG rect inject (old bg detection): Injects a rect into the SVG clone but still uses the broken bg detection
  3. Fix B: Canvas fillRect (old bg detection): Uses fillRect on canvas but still uses the broken bg detection
  4. Fix C: Smart bg detection + SVG rect inject: Only considers rects that cover ≥90% of the SVG area as background rects. Falls back to #ffffff. Injects the detected bg as a rect into the SVG clone.
